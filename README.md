# 🚀 Real-Time ML Feature Engineering Pipeline

A production-style real-time feature engineering pipeline built using **Apache Kafka**, **Apache Flink**, **Python**, **Docker**, and **Streamlit**. The pipeline simulates user interactions, processes streaming data using event-time processing and watermarks, computes machine learning features in real time, and provides observability through a dashboard.

---

## 📌 Project Overview

Modern machine learning systems require fresh features instead of stale batch-computed data. This project demonstrates how to build a real-time streaming pipeline capable of computing user and content features with low latency.

The pipeline performs:

* Real-time user event processing
* Event-time processing using Apache Flink
* Watermark handling for late events
* Tumbling and Sliding Window computations
* Kafka-based event streaming
* Feature engineering for ML systems
* Dashboard for monitoring pipeline activity

## 📊 Kafka Topics

### user-events

Stores simulated user interaction events.

Schema

```json
{
  "user_id": "string",
  "content_id": "string",
  "event_type": "string",
  "dwell_time_ms": 5000,
  "timestamp": "ISO-8601"
}
```

---

### content-metadata

Stores metadata for each content item.

```json
{
  "content_id": "movie_1",
  "category": "SciFi",
  "creator_id": "creator_1",
  "publish_timestamp": "ISO-8601"
}
```

---

### feature-store

Stores engineered ML features.

```json
{
  "entity_id": "user_1",
  "feature_name": "click_rate",
  "feature_value": 0.25,
  "computed_at": "ISO-8601"
}
```

## ⚙️ Setup

Clone the repository

```bash
git clone https://github.com/Swetha-svvv/realtime-ml-feature-pipeline.git
```

Move into the project

```bash
cd realtime-ml-feature-pipeline
```

Build the containers

```bash
docker compose build
```

Start all services

```bash
docker compose up -d
```

---

## ▶️ Running the Flink Job

Enter the Flink container

```bash
docker exec -it flink-job bash
```

Run the application

```bash
python app.py
```

## ⏱️ Event Time & Watermarks

The pipeline processes events using Event-Time semantics.

Watermark Strategy:

* Bounded Out-of-Orderness
* 30 Seconds

Late events generated by the producer are handled correctly within the watermark tolerance.
